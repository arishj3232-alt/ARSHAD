Build a full-stack, production-level 2-user private communication web application called "OnlyTwo".

This must be a REAL working system (not a demo), including frontend, backend, real-time communication, persistent storage, media handling, and call functionality.

────────────────────────
🎯 CORE REQUIREMENTS
────────────────────────

* Only 2 users allowed in the system
* No signup/login pages
* Entry via secret room code
* Block any third user with message:
  "This space is only for two"
* Maintain persistent session

────────────────────────
⚙️ TECH STACK
────────────────────────
Frontend:

* React (modern component-based)
* Tailwind CSS for styling

Backend:

* Node.js + Express

Database:

* Firebase Firestore (for messages)

Storage:

* Firebase Storage (for images, videos, audio)

Realtime:

* Firebase real-time listeners (onSnapshot)

Calls:

* WebRTC (peer-to-peer)
* Use backend only for signaling

────────────────────────
💬 CHAT SYSTEM
────────────────────────

* Real-time messaging (instant updates)
* Text messages with emoji support
* Reply to message
* Seen / delivered status
* Typing indicator
* Smooth UI updates
* Auto-scroll to latest message

────────────────────────
📜 SMART MESSAGE LOADING
────────────────────────

* Load only last 20 messages initially
* Implement pagination (load older messages on scroll)
* Maintain scroll position while loading

────────────────────────
🗑️ DELETE FOR EVERYONE
────────────────────────

* Delete message for both users
* Do not remove from DB, instead mark as deleted
* Replace UI with:
  "This message was deleted"

────────────────────────

────────────────────────

* Upload and send images

* Inline video player

────────────────────────
🎤 VOICE MESSAGE SYSTEM
────────────────────────

────────────────────────
